What is Container Storage Modules?

The Dell Technologies (Dell) Container Storage Modules (CSM) enables simple and consistent integration and automation experiences, extending enterprise storage capabilities to Kubernetes for cloud-native stateful applications. It reduces management complexity so developers can independently consume enterprise storage with ease and automate daily operations such as provisioning, snapshotting, replication, observability, authorization, application mobility, encryption, and resiliency.



{{% cardpane %}} {{< card header="[**Authorization**](docs/concepts/authorization/)">}} CSM for Authorization provides storage and Kubernetes administrators the ability to apply RBAC for Dell CSI Drivers. It does this by deploying a proxy between the CSI driver and the storage system to enforce role-based access and usage rules.
{{< /card >}} {{< card header="[**Replication**](docs/concepts/replication/)">}} CSM for Replication project aims to bring Replication & Disaster Recovery capabilities of Dell Storage Arrays to Kubernetes clusters. It helps you replicate groups of volumes and can provide you a way to restart applications in case of both planned and unplanned migration. {{< /card >}} {{< card header="[**Resiliency**](docs/concepts/resiliency/)">}} CSM for Resiliency is designed to make Kubernetes Applications, including those that utilize persistent storage, more resilient to various failures. {{< /card >}} {{% /cardpane %}} {{% cardpane %}} {{< card header="[**Observability**](docs/concepts/observability/)">}} CSM for Observability provides visibility on the capacity of the volumes/file shares that is being managed with Dell CSM CSI (Container Storage Interface) drivers along with their performance in terms of bandwidth, IOPS, and response time. {{< /card >}} {{< card header="[**Application Mobility**](docs/concepts/applicationmobility/)">}} Container Storage Modules for Application Mobility provide Kubernetes administrators the ability to clone their stateful application workloads and application data to other clusters in the cloud. {{< /card >}} {{< card header="[License](support/license/)" >}} The tech-preview releases of Encryption require a license.
